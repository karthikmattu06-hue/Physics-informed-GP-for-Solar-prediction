# **Task 2: Human Activity Recognition (HAR) System**

This project implements a deployment-ready human activity recognition pipeline using an **XGBoost Classifier** integrated with a **SMOTE** (Synthetic Minority Over-sampling Technique) pipeline. The system uses a standalone prediction script to process anonymous sensor data and identify various physical activities (including rare classes like Jogging and Pushups) with high macro-average F1 performance.

## **System Overview**

The core of the deployment is `predict_task2.py`, which:

1.  **Loads Pre-trained Models:** Imports the frozen XGBoost pipeline and label encoders generated by the training notebook.
2.  **Performs Feature Engineering:** Replicates the physics-based feature extraction used during training (e.g., Interaction Gates, Rotational Energy Ratios).
3.  **Generates Predictions:** Outputs class labels (e.g., 'Jogging', 'WalkingUpstairs') for every input row.

## **Required Artifacts**

For the prediction script to function, the following pre-trained artifact files must be present in the same directory as `predict_task2.py`. These represent the "frozen" state of the system and are generated by running **`task_2.ipynb`**:

## **Usage**

To evaluate the model on a new dataset, run the script from the command line.

### **Command Syntax**

Bash

```
python predict_task2.py <path_to_csv>
```

### **Example**

Bash

```
python predict_task2.py har_train.csv
```

### **Input Data Format**

The input CSV must follow the **Standard HAR Dataset Format** (no header), specifically:

* **Structure:** No header row.  
* **Features:** Columns 0 to 327 contain pre-computed numerical sensor features (float).  
* **Target (Optional):** The last column (Column 328\) contains the Activity Label (e.g., 'Sitting'). If present, the script will calculate and print evaluation metrics.

### **Output**

1. **Submission File:**  
   * The script **always** generates a file named `task2_predictions.csv` in the current directory.  
   * This file mirrors the **exact format** of the input CSV (retaining all original columns).  
   * The last column is overwritten (or added) containing the model's **predicted class labels**, replacing any ground truth labels.  
   * This is the file required for the assignment submission.  
2. **Console Metrics:**  
   * If the input file contains ground truth labels (i.e., it has the expected number of columns), the script will also print:  
     * **Confusion Matrix** (showing counts for all classes).  
     * **Accuracy Score**.  
     * **Macro-Average F1 Score**.

## **Methodology**

The model was developed to solve the specific challenges of this dataset:

1. **Physics-Based Feature Engineering:**  
   * **Stairs\_Ratio:** We engineered a custom feature `Col_24 / Col_30` to amplify the difference between "Walking" and "WalkingUpstairs," which overlap significantly in the raw feature space.  
   * **Jog\_Signal:** An interaction feature `Col_159 * Col_30` was created to identify the high-variance signature of "Jogging," allowing the model to detect it despite having only \~12 training samples.  
2. **Imbalanced Learning Strategy:**  
   * The pipeline integrates **SMOTE** (k\_neighbors=3) to synthetically upsample rare classes (Jogging, Pushups) during training.  
   * This prevents the model from ignoring minority classes to maximize accuracy, ensuring a high **Macro-F1 score**.
